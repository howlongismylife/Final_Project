В директории `tests` находятся тесты для проверки API, которое должно быть реализовано в веб-сервере.

Директория `web` содержит файлы фронтенда.

# TODO Planner (Golang)

## Описание проекта
Веб-приложение для управления задачами.  
Позволяет создавать, редактировать, удалять задачи, а также отмечать их выполненными.  
Поддерживаются повторяющиеся задачи с автоматическим пересчётом даты.

Проект реализован в рамках итогового задания. Backend написан на Go, используется SQLite.



## Функциональность

Реализованы следующие API-методы:

- POST /api/task — создание задачи
- GET /api/tasks — получение списка задач
- GET /api/task?id= — получение одной задачи
- PUT /api/task — обновление задачи
- DELETE /api/task?id= — удаление задачи
- POST /api/task/done?id= — отметка задачи выполненной

Дополнительно:
- обработка повторяющихся задач
- валидация входных данных
- сортировка задач по дате



## Аутентификация

Добавлена простая авторизация по паролю.

Пароль задаётся через переменную окружения:

TODO_PASSWORD=12345

Авторизация выполняется через:

POST /api/signin


При успешном входе сервер возвращает token, который используется в cookie.



## Запуск проекта локально

1. Установить переменную окружения:

export TODO_PASSWORD=12345

2. Запустить сервер:


go run main.go

3. Открыть в браузере:


http://localhost:7540


## Тестирование

Запуск всех тестов:


go test ./tests


Если включена аутентификация, необходимо указать token в файле:


tests/settings.go

### Сборка образа


docker build -t todo-app .


### Запуск контейнера


docker run -p 7540:7540 -e TODO_PASSWORD=12345 todo-app


### Запуск с базой данных


docker run -p 7540:7540 \
-e TODO_PASSWORD=12345 \
-e TODO_DBFILE=/data/scheduler.db \
-v $(pwd):/data \
todo-app
